## Ratified Atomic Concerns (2026-05-06)

Six Concerns at atomic-worry altitude — each names a real worry without encoding mechanism. NCs and Evidence anchor here; RCs ratify against these and only these.

### C-1 — Sufficient seed information

Skill must start with sufficient seed information to build shared understanding.

### C-2 — Designer consumption of proof state

Designer must be able to consume proof state to build shared understanding, including problem-space coordinate placement.

### C-3 — Proof state faithfulness

Proof state must remain faithful to designer's intent over time.

### C-4 — Designer-consent gate on advancement

Proof advancement without designer consent breaks shared understanding.

### C-5 — Specify handoff fidelity

Specify handoff must carry designer's intent with sufficient fidelity to produce correct architecture.

### C-6 — Mutual agreement on specify-input adequacy

Agent and designer must mutually agree that the information provided to design-specify produces the correct architecture.

## Derived NC Set (Pending Authoring)

The reframing surfaces a starter NC set spanning fifteen NCs across the six Concerns. These are not yet authored as proof elements — they remain in this working record until structured authoring happens during the Round Cycle.

**Anchored to C-1:**
- Initial topic and concerns derived from invocation context plus parallel exploration before proof open.
- Parallel context exploration must run before the round cycle.

**Anchored to C-2:**
- Round-trip readability: proof state readable directly, no separate translation pass.
- Information packet structure (insight + intro + facts + agent commentary + pessimist commentary + decision).
- PM register applies to all designer-facing output (Translation Gate).
- Single topic per round.
- Problem-space coordinate placement is a tracked proof-state property, visible in every read.

**Anchored to C-3:**
- Vocabulary discipline structural, not aspirational.
- Authority visible on every claim.
- Element-type set is closed; extension is charter-level.
- Every advancement carries provenance.
- Withdrawal grammar preserves element with disposition; never silent deletion.
- CRUD-completeness across the closed element set (per `cluster-d-proof-layer-crud-requirements.md`).
- Granularity discipline — atomic single-claim elements.
- Round topic selection uses proof gaps as input.

**Anchored to C-4:**
- Agent may not advance the proof without designer ratification of the change.

**Anchored to C-5:**
- Design brief at closure carries complete proof envelope.
